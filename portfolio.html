DiceBag

A web app designed to augment the traditional table top RPG gameplay. User is able to create custom dice and have them saved locally in the browser. the app handles the rolls from there with a simple click of the dice. I thought this would help speed up combat and other time consuming parts of Dungeons and Dragons where math needs to be done on the fly and often times can lead to mistakes being made and time wasted. This was a project I did for myself and for fun off to the side while attending UT's Fullstack bootcamp.

Hangman

The next technology to be utilized was Javascript. Sites could now take in more information, process it, and dynamically change based on that. Using input and listeners, I was able to take the keys being pressed and compare them against an array and with that simple comparison create a small game.

Blackest Dungeon: basic rpg

This game was to demonstrate my evolving Javascript abilities. The data being tracked became more robust, more numbers and function calls. I implimented a very crude version of a d20 system such as Pathfinder to determine if my heros and monsters would hit, and rolled their damage in a similar fashion.

Get OUT!

A choose your own adventure game with a demonstration on using timer functions in Javascript. The goal is to explore the house and escape before the clock strikes midnight. I played around with much larger data structures and allowing Javascript to dynamically draw the user interface depending on certain flags inside the code. The data structures ended up becoming cumbersome, and didnt allow for easy scaling and creating more rooms.

Essentals: what's in your area?

A group project that allows the user to search their area and get back specific information about the stores and other facilities in that area. the catagories returned are: hospitals, libraries, grocery stores, gas stations, and post offices. This was created to demonstrate using multiple APIs, and using all of their data to create a working product.
